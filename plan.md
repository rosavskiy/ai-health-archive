ТЕХНИЧЕСКОЕ ЗАДАНИЕ (ТЗ) НА РАЗРАБОТКУ PWA-СЕРВИСА «AI HEALTH ARCHIVE» (ВЕРСИЯ 5.0 — PWA EDITION)

1. ОБЩАЯ КОНЦЕПЦИЯ И СТЕК (PWA/BACKEND)
1.1. Форм-фактор: Progressive Web App (PWA). Поддержка Service Workers для работы offline, манифест для установки на экран «Домой», Push-уведомления.
1.2. Frontend: React.js или Next.js + Tailwind CSS. Адаптив под мобильные устройства (Mobile First).
1.3. Backend: Python (FastAPI) — оптимально для интеграции с ML/AI библиотеками.
1.4. База данных: PostgreSQL (хранение структурированных медицинских данных).
1.5. Инфраструктура: Полное развертывание в РФ-контуре (Yandex Cloud / Selectel).

2. ЮРИДИЧЕСКИЙ КОНТУР (БЕЗОПАСНОСТЬ И 152-ФЗ РФ)
2.1. Локализация: Хранение всех персональных данных и медицинских записей строго на серверах внутри РФ.
2.2. Правовой комплаенс: При первом входе в PWA — обязательный экран с Акцептом оферты, Политикой конфиденциальности и Согласием на обработку специальных категорий ПД (медицина) и их деперсонализацию.
2.3. Трансграничный запрет: Передача связки «Личность + Медицинские данные» за пределы РФ запрещена. Иностранные LLM (OpenAI/Claude) используются только как «процессоры» для анонимизированного текста.

3. ПРОТОКОЛ МАСКИРОВАНИЯ (PII MASKING & ANONYMIZATION)
3.1. Модуль «AI-Shield»: Автоматическая очистка всех входящих документов от персональных данных (PII) перед индексацией.
3.2. Алгоритм обработки:

Шаг 1: OCR (Yandex Vision) извлекает текст из фото/PDF.

Шаг 2: ИИ-агент (NER) находит в тексте: ФИО, адрес, СНИЛС, полис, телефон.

Шаг 3: Маскирование — в БД сохраняется текст, где эти данные заменены на токены (например, [USER_ID_MASK]).

Шаг 4: Графическое маскирование — при генерации превью документа в PWA на личные данные накладывается визуальный слой (Blur/Black bar).
3.3. Результат: В аналитическом модуле хранятся только чистые медицинские показатели, не привязанные к ФИО внутри самого текста.

4. ФУНКЦИОНАЛЬНЫЕ ВОЗМОЖНОСТИ PWA
4.1. Авто-сбор (Email Sync): Фоновый сервис (Celery/n8n) подключается к почте через IMAP. При поступлении письма от лабораторий (Invitro, Helix и др.) вложения автоматически улетают в «AI-Shield» и далее в личный кабинет.
4.2. Ручная загрузка: Кнопка «Добавить анализ» в PWA с доступом к камере смартфона или галерее.
4.3. Личный кабинет (Health Dashboard):

Секция «Архив»: Список всех документов с поиском.

Секция «Динамика»: Интерактивные графики (Chart.js) по ключевым показателям (холестерин, сахар, витамины).

Секция «AI-Консультант»: Чат-интерфейс, где можно спросить: «Покажи динамику железа» или «Что значит рост показателя X в моих последних трех анализах?».
4.4. Push-уведомления: «Обнаружены новые анализы в вашей почте», «Пора сдать плановый анализ крови».

5. ТЕХНИЧЕСКАЯ БЕЗОПАСНОСТЬ
5.1. Шифрование: Хранение оригиналов документов в зашифрованном S3-хранилище (AES-256).
5.2. Авторизация: Двухфакторная аутентификация (2FA) или вход по защищенной ссылке/OTP.
5.3. Анонимность в БД: Связь между пользователем и его медицинскими данными осуществляется через зашифрованный внутренний ID (UUID), без использования ФИО в таблицах показателей.

6. ИНСТРУКЦИЯ ДЛЯ AI-АГЕНТА (ПЕРВЫЕ ШАГИ)

Разработай схему базы данных PostgreSQL, поддерживающую деперсонализированное хранение (Users <-> Encrypted_Docs <-> Metrics).

Напиши FastAPI роут для загрузки файла, который вызывает цепочку: OCR -> LLM Masking -> Data Extraction.

Создай манифест PWA (manifest.json) и базовый Service Worker для фронтенда.

Реализуй скрипт для подключения к Gmail/Yandex через IMAP для поиска PDF-вложений.
